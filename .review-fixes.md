## Issues Found & Fixed

### Issue #1: Incorrect Function Signature for AddLLMAttributes ✅ FIXED
**Problem:** Spec required `AddLLMAttributes(span trace.Span, attrs map[string]interface{})` but implementation used SDK-specific `sdktrace.ReadWriteSpan`.

**Fix:**
- Changed signature to use standard `oteltrace.Span` interface
- Removed dependency on `go.opentelemetry.io/otel/sdk/trace` in otel_attributes.go
- Updated logger.go to remove `ReadWriteSpan` type assertions

**Files Modified:**
- `server/trace/otel_attributes.go` - Updated signature and imports
- `server/trace/logger.go` - Removed type assertions, cleaned up imports

---

### Issue #2: AddLLMAttributes Only Added LLM-Specific Attributes ✅ FIXED
**Problem:** Spec says "OTEL span should include all internal span attributes as OTEL attributes" but implementation only added specific LLM keys.

**Fix:**
- Completely rewrote `AddLLMAttributes()` to iterate through all attributes in the map
- Uses type switch to convert all Go types to appropriate OTEL attribute types
- Now adds ALL attributes, not just those matching LLM constants

**Implementation:**
```go
for key, value := range attrs {
    switch v := value.(type) {
    case string:
        otelAttrs = append(otelAttrs, attribute.String(key, v))
    case int:
        otelAttrs = append(otelAttrs, attribute.Int64(key, int64(v)))
    // ... handles all types
    }
}
```
