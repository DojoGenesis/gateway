## Vision & Purpose

**One-Sentence Mission:**
A modular Go framework for building agentic AI systems with pluggable providers, tool execution, DAG-based orchestration, persistent memory, a tiered skill system, and MCP Apps host infrastructure.

**Core Principles:**
- **Interface-driven architecture** — All major components expose clean interfaces for testing and extensibility
- **Disposition-aware execution** — Gateway-ADA Contract compliance shapes pacing, depth, tone, and initiative
- **Observable by default** — OTEL tracing throughout, Langfuse integration for production monitoring
- **Standalone modularity** — Each module can be imported independently (orchestration, skill, memory, mcp, tools, apps)

## What's New in v1.1.0

### MCP Apps Bridge — The App Bridge

Added MCP Apps host infrastructure (beta) as a new `apps/` module with 6 HTTP endpoints:

**Components:**
- `ResourceRegistry` — `ui://` resource storage with ETag/304 caching
- `AppRegistry` — App instance lifecycle (launch/close/list)
- `AuthorizationPolicy` — Tool-level ACL with wildcard grants
- `ToolCallProxy` — Proxied tool calls through gateway's ToolRegistry
- `SecurityPolicy` — CSP headers, permission validation, dangerous permission blocking
- `AppManager` — Top-level coordinator wiring all components
- `MCPResourceLoader` — Loads resources from MCP servers via `ReadResource`

**Infrastructure Changes:**
- `MCPServerConnection.ReadResource()` added to `mcp/connection.go`
- 6 new handlers in `server/handle_apps.go`
- 6 new routes in `server/router.go` (51 total, up from 45)
- `MCPAppsConfig` in `server/config/config.go`
- Feature flag: `MCP_APPS_ENABLED` (default: false)
- `AppManager` field in `server/deps.go` and `server/server.go`
- Version bumped to `1.1.0`

**Quality:**
- 59 tests (unit + integration + benchmarks)
- 98.1% coverage
- Race-detection clean
- `go vet` clean
